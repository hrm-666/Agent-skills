## 使用方式

- [ ] 严格按 Phase 推进，不跨 Phase 偷跑功能
- [ ] 每完成一个 Phase 就暂停，先让用户手动验收
- [ ] 优先级始终保持：正确性 > 极简 > 优雅 > 功能
- [ ] 任何会扩大范围的改动先单列，不混入主线实现

## 全程不可违背的约束

- [ ] 内置工具始终只有 4 个：`read / write / bash / activate_skill`
- [ ] Agent 保持无状态；每次请求独立，不做历史持久化
- [ ] 核心 system prompt 保持短小，skill 正文按需加载
- [ ] 核心引擎只负责 LLM 调度、工具执行、skill 加载
- [ ] 不在核心里写死数据库、OCR、HTTP 等业务概念
- [ ] 全部使用同步流程，不引入 async、thread、process 并发
- [ ] 不引入 langchain / llamaindex / agno 等框架
- [ ] WebUI 不引入前端构建工具
- [ ] 代码注释用中文，变量名和函数名用英文
- [ ] 所有工具调用、LLM 调用、错误都进入日志
- [ ] 最终代码总行数控制在 1000 行以内（不含空行和注释）

## 实现前先对齐的分叉点

- [ ] 确认项目代码是直接落在当前目录，还是新建 `mini-agent/` 子目录后再实现
- [ ] 确认是否单独创建 `workspace/` 目录，以满足 `write` 工具的安全边界
- [ ] 确认 skill 扫描范围是否只做 `./skills/`，还是兼容 `./.agents/skills/`
- [ ] 确认 skill 校验策略是严格按 `Task6.md` 执行，还是兼容 agentskills 建议的 lenient 加载

## 默认建议实现策略

- [ ] 当前工作区直接作为项目工作根目录，不额外嵌一层复杂结构
- [ ] 单独创建 `workspace/`，并将 `write` 限制在 `workspace/`、`uploads/`、`logs/`
- [ ] MVP 只扫描 `./skills/`，不额外扩展用户级 skill 目录
- [ ] skill 校验以 `Task6.md` 的严格规则为准
- [ ] `compatibility`、`metadata` 可以保留解析
- [ ] `allowed-tools` 先不做执行逻辑，只作为未来兼容位

## 从 agentskills-main 吸收的实现细则

- [ ] 启动时只加载每个 skill 的 `name + description`，不预读完整正文
- [ ] 只有在模型判断需要时，才通过 `activate_skill` 加载对应 `SKILL.md` 正文
- [ ] `scripts/`、`references/`、`assets/` 不预加载，按正文需要再读
- [ ] skill 内引用其他文件时，统一使用相对 skill 根目录的路径
- [ ] 示例 skill 的 `SKILL.md` 尽量保持精炼，避免把参考资料全塞正文
- [ ] skill 中的脚本必须非交互、可重复执行、错误信息明确
- [ ] skill 脚本尽量输出结构化结果，优先 JSON 或稳定文本格式
