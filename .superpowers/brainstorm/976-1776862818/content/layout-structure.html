布局结构偏好

极简专业风格下，如何组织聊天界面的空间？

既然选择了极简专业风（类似 Linear、Vercel），接下来需要确定布局结构。考虑到你提到的信息密度和响应式问题，我准备了几种布局方案：

顶部导航
对话区域（全宽）
消息、工具步骤都在这里
输入框

单列布局

所有内容在一个主列中，工具步骤内联在对话流中。类似 ChatGPT 的布局。

✓ 响应式最简单
✓ 移动端体验好
✗ 工具步骤可能打断对话流
顶部导航
对话区域
工具面板
步骤、状态

右侧边栏

对话在左，工具步骤和状态信息在右侧独立面板。类似当前设计但更精简。

✓ 信息分离清晰
✓ 可以同时看对话和工具
✗ 小屏幕需要折叠
顶部导航 + 标签切换
当前视图
对话 | 工具 | 设置（标签切换）

标签切换

通过顶部标签在"对话"、"工具监控"、"设置"之间切换，每次只显示一个视图。

✓ 信息密度最低
✓ 响应式友好
✗ 需要切换才能看工具
顶部导航
对话区域（全宽）
工具抽屉（可展开/收起）

底部抽屉

对话占据主要空间，工具步骤在底部可展开的抽屉中。类似 VS Code 的终端面板。

✓ 对话区域最大化
✓ 工具信息按需查看
✗ 需要额外操作展开

提示

考虑你的使用场景：如果经常需要查看工具调用细节，选右侧边栏或底部抽屉；如果更关注对话本身，选单列或标签切换。
