# mini-agent — 多人协同开发工程

本工程是一个**多人协同使用**的项目框架。每位成员在根目录下创建自己命名的文件夹，将各自的代码放在自己的文件夹内，互不干扰。

## 目录结构说明

```
mini-agent/
├── mhr/            # 成员 马皓然 的工作目录
│   ├── core/       # 核心模块（Agent loop、LLM 调用等）
│   ├── adapters/   # CLI 与 FastAPI 入口
│   ├── tools_builtin/  # 内置工具
│   ├── skills/     # 技能目录
│   ├── webui/      # 前端页面
│   ├── data/       # 示例数据脚本
│   ├── uploads/    # 上传目录
│   ├── logs/       # 日志目录
│   ├── .env        # 个人环境变量（已加入 .gitignore，不会提交）
│   ├── .env.example    # 环境变量模板
│   └── config.yaml     # 个人配置文件
├── <你的名字>/     # 你的工作目录（参照 mhr/ 的结构）
├── README.md
├── .gitignore
└── ...
```

## 协同开发流程

1. **创建个人目录**：在根目录下创建以自己名字（或昵称）命名的文件夹，例如 `zhangsan/`
2. **独立开发**：在个人目录下拥有完整的项目副本，包含 `core/`、`adapters/`、`skills/` 等所有子目录
3. **配置隔离**：每个人的 `.env` 和 `config.yaml` 都在自己的目录下，互不影响
4. **互不干扰**：每个人的代码独立演进，不会因为他人修改而产生冲突

> 参考 [`mhr/`](./mhr/) 目录的结构来搭建你的个人工作目录。

## 安全说明

- `.env` 文件已加入 `.gitignore`，包含 API Key 等敏感信息不会被提交到 Git
- 提交前请确认没有在代码或配置文件中硬编码密钥
- 每个成员需自行复制 `.env.example` 为 `.env` 并填入自己的 API Key
