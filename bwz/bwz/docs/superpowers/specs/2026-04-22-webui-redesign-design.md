# Mini Agent WebUI 重新设计方案

**日期：** 2026-04-22  
**设计师：** Claude  
**状态：** 待审阅

## 1. 设计目标

基于用户反馈，重新设计 Mini Agent 的 WebUI 界面，解决以下问题：

- **视觉风格不够现代** - 配色、字体、圆角等视觉元素感觉过时或不够精致
- **信息密度问题** - 当前界面过于空旷，浪费空间
- **响应式设计不够好** - 在不同屏幕尺寸下表现不佳
- **整体感觉不对** - 缺少专业感和简洁感

## 2. 设计方向

### 2.1 设计风格

**极简专业风**（类似 Linear、Vercel）

- 克制、高效、工具感
- 大量留白、中性配色
- 清晰的信息层级
- 专注于功能本身
- 适合开发者和专业用户

**为什么选择这个方向：**
- 符合 Agent 工具的定位（开发工具，非消费级产品）
- 长时间使用不会产生视觉疲劳
- 专业感强，信任度高
- 易于维护和扩展

### 2.2 布局结构

**右侧边栏布局**

```
┌─────────────────────────────────────────────────┐
│ 顶部导航栏（固定，64px）                          │
├──────────────────────────┬──────────────────────┤
│                          │                      │
│  对话区域（60%）          │  工具面板（40%）      │
│  - 消息列表              │  - 工具步骤流         │
│  - 输入框（底部固定）     │  - 实时状态          │
│                          │                      │
└──────────────────────────┴──────────────────────┘
```

**布局特点：**
- 对话区域在左侧，占据主要空间（60%）
- 工具步骤和状态信息在右侧独立面板（40%）
- 信息分离清晰，可以同时查看对话和工具调用
- 避免工具步骤打断对话流

### 2.3 配色方案

**暖灰 + 深蓝灰**（类似 Vercel）

```
背景色：    #fafaf9  （极浅暖灰）
表面色：    #f5f5f4  （浅暖灰）
边框色：    #e7e5e4  （中暖灰）
文字主色：  #0f172a  （深蓝灰）
文字次色：  #78716c  （石板灰）
强调色：    #0f172a  （深蓝灰）
成功色：    #059669  （绿色）
错误色：    #dc2626  （红色）
```

**为什么选择暖灰：**
- 比冷灰更温暖柔和，减少冷感
- 长时间使用更舒适，不刺眼
- 保持专业感的同时增加亲和力
- 与深蓝灰强调色搭配和谐

### 2.4 信息密度

**宽松舒适（低密度）**

```
消息气泡间距：  24-32px
内边距：        20-24px
字号：          16-18px（正文）
圆角：          16-24px
行高：          1.6-1.8
```

**为什么保持宽松：**
- 用户选择保持当前的宽松风格
- 适合长时间阅读和使用
- 不会感觉拥挤或压抑
- 符合极简专业风的设计语言

### 2.5 响应式策略

**可折叠抽屉**

- **桌面（>1024px）：** 左右分栏，工具面板固定显示
- **平板（768-1024px）：** 可折叠侧边栏，按钮切换显示/隐藏
- **手机（<768px）：** 底部抽屉，点击浮动按钮展开

**为什么选择这个策略：**
- 桌面端保持最佳体验（同时看对话和工具）
- 移动端对话区域最大化
- 工具信息按需查看，不占用主要空间
- 比标签切换更灵活，比堆叠更节省空间

## 3. 视觉规范

### 3.1 字体系统

```
字体族：Inter, -apple-system, BlinkMacSystemFont, sans-serif

标题 H1：  24px / 600 / -0.02em
标题 H2：  20px / 600 / -0.01em
标题 H3：  18px / 600 / normal
正文：     16px / 400 / 1.6
小字：     14px / 400 / 1.5
标签：     12px / 600 / 0.05em（大写）
```

**字体选择理由：**
- Inter 是现代极简设计的标准字体
- 可读性强，数字和字母区分度高
- 支持中文回退到系统字体
- 字重 400/600 足够，避免过多变化

### 3.2 间距系统

```
xs:   4px   （图标间距、小元素）
sm:   8px   （紧凑间距）
md:   16px  （标准间距）
lg:   24px  （段落间距）
xl:   32px  （区块间距）
2xl:  48px  （大区块间距）
```

### 3.3 圆角系统

```
sm:   8px   （按钮、输入框）
md:   12px  （卡片、小面板）
lg:   16px  （对话气泡、大面板）
xl:   20px  （容器）
full: 9999px（圆形按钮）
```

**圆角策略：**
- 保持一致的圆角比例
- 大元素用大圆角，小元素用小圆角
- 避免过度圆润（不超过 24px）

### 3.4 阴影系统

```
soft:   0 2px 8px rgba(15, 23, 42, 0.04)   （轻微阴影）
panel:  0 4px 16px rgba(15, 23, 42, 0.06)  （面板阴影）
float:  0 8px 24px rgba(15, 23, 42, 0.08)  （浮动元素）
```

**阴影使用原则：**
- 极简风格，阴影要轻
- 只在需要层级区分时使用
- 避免多层阴影堆叠

## 5. 交互设计

### 5.1 微交互

**按钮：**
- Hover：scale(0.98)，过渡 150ms
- Active：scale(0.95)，过渡 100ms
- Focus：outline 2px #0f172a，offset 2px

**输入框：**
- Focus：边框加粗（1px → 2px），颜色变化
- 过渡：border 150ms ease-out

**卡片：**
- Hover：阴影加深（soft → panel）
- 过渡：box-shadow 200ms ease-out

**抽屉：**
- 展开：transform translateY(0)
- 收起：transform translateY(100%)
- 过渡：300ms ease-out
- 背景遮罩：rgba(0, 0, 0, 0.3)，过渡 200ms

### 5.2 加载状态

**发送消息中：**
- 发送按钮：文字变为"处理中..."，禁用
- 输入框：禁用
- Provider 选择器：禁用
- 对话区：显示占位气泡（Agent 消息样式，文字"正在处理..."）

**占位气泡动画：**
- 三个点，依次淡入淡出
- 动画时长：1.5s，无限循环
- 颜色：#78716c

**上传文件中：**
- 附件标签：显示进度条
- 进度条：高度 2px，背景 #0f172a
- 动画：宽度从 0% 到 100%

### 5.3 错误处理

**错误横幅：**
- 位置：顶部导航栏下方
- 背景：#fef2f2
- 文字：#dc2626
- 边框：底部 1px #fecaca
- 圆角：0（全宽）
- 内边距：12px 24px
- 字号：14px
- 关闭按钮：右侧，图标按钮

**错误消息气泡：**
- 背景：#fef2f2
- 文字：#dc2626
- 边框：1px #fecaca
- 其他样式同 Agent 消息

### 5.4 滚动行为

**对话区域：**
- 新消息自动滚动到底部
- 用户手动滚动时暂停自动滚动
- 滚动到底部后恢复自动滚动
- 滚动条：宽度 8px，颜色 #e7e5e4

**工具面板：**
- 新步骤添加到顶部（prepend）
- 不自动滚动
- 滚动条：同对话区域

### 6.1 断点定义

```
mobile:  < 768px
tablet:  768px - 1024px
desktop: > 1024px

```

### 6.2 布局变化

**Desktop (>1024px)：**
- 左右分栏，比例 60:40
- 工具面板固定显示
- 导航栏全功能

**Tablet (768-1024px)：**

- 工具面板变为底部抽屉
- 浮动按钮触发
- 导航栏简化（隐藏部分控件）

- 统一的间距系统
